A schematic is a list of lines — components and nets. Lines starting with #, // or %% are comments; a lone schematic line is ignored.

1 · Header directives

Theme: blueprintset the canvas theme from the script (light · dark · blueprint · mono)
Title: "My board"document title (metadata)

2 · Components

<ID>: <name> <subname> [@x,y] [rot:N] [flip] [flipv] [*w,h] [#hex] [bg:#hex]

ID is the designator you reference in nets (e.g. R1, U2). name is the component type from the catalog (resistor, lm358, led…). subname is the value/label shown under it (10k, +5V, BC547).

R1: resistor 10k @12,4resistor R1, value 10k, pinned at cell 12,4
U1: lm358 @24,6an IC (no value)
C1: capacitor rot:90orientation — rot:0|90|180|270
U2: lm358 @24,6,90orientation as the 3rd value of @x,y,deg
Q1: npn BC547 flipmirror horizontally
Q2: npn BC547 flipvmirror vertically (alias mirrorv/vflip)
J1: header *1,8resize a flexible part — *width,height in cells. Only flexW/flexH parts honor it; here height 8 → a 4-pin header (1 pin per 2 cells), the width is ignored
D1: led #2266ffoverride the symbol color (hex)
U1: lm358 bg:#3345aaffbackground fill — 8-digit hex #rrggbbaa adds transparency for a highlight tint

The @x,y cell is the top-left of the part's box; pins sit at fixed offsets from it. Wrap a value in quotes only if it contains spaces, e.g. U1: lm358 "dual amp".

3 · Nets (wires between pins)

Reference a pin as ID:pin (or ID.pin). A net lists every pin on the same node; junction dots appear automatically where three or more meet.

net: SIG R1:2 U1:3connect two pins, net named SIG
net: +5V PWR:+ U1:8 R1:1multi-pin power net
wire: U1:1 D1:aa single unnamed wire
net: +5V #d23 PWR:+ R1:1colour the whole net (hex, anywhere on the line) — wires & junction dots
label: @40,2 "OUT"a free net label at a grid cell

Colour overrides: add a #hex to a component line to recolour its symbol (D1: led #2266ff), or to a net:/wire: line to recolour that net's traces — handy for highlighting power, ground or a signal of interest.

schematic
  Theme: blueprint
  PWR: vcc +5V  @8,0
  R1:  resistor 330 @10,4
  D1:  led @20,3
  G1:  gnd @24,8
  net: VCC PWR:+ R1:1
  net: SIG R1:2 D1:a
  net: OUT D1:c G1:gTry ▸

4 · Pin names by part

Two-terminal parts use 1/2 (or a/c for diodes/LEDs). Transistors use b/c/e. ICs & modules use their printed pin names (U1:OUT, U1:+IN) or numbers (U1:8). Op-amp triangles add V+/V-/out. Power parts expose + (vcc) and g (gnd).

5 · Wiring & auto-routing

Every wire leaves a pin through a one-cell standoff (so lines never run across the pins or the body), then an A* grid router finds an orthogonal path that goes around parts and penalizes turns and crossings with other nets. Placement still matters — pin via @x,y to lay things out; the router handles the connections.

6 · Themes

Use Theme: in the script, the canvas selector above, or the API. Every visual element is its own token:

new SCH.Schematic('#app', { theme: 'blueprint' });

// override individual tokens (merged onto a base preset):
new SCH.Schematic('#app', { theme: {
  base: 'dark',
  part:'#e0a458',   // component leads / outline
  body:'#0f141a',   // body fill
  wire:'#4cc38a',   // wires
  junction:'#4cc38a',
  pin:'#5aa9e6',    // pin labels & leads
  ref:'#e0a458',    // designators (R1, U2)
  value:'#aeb9c4',  // values (10k)
  name:'#e6edf3',   // IC / module name
  bg:'#0f141a', grid:'#1b232c'
}});

7 · JavaScript API

const s = new SCH.Schematic('#app', { cols:60, rows:36, cell:16, theme:'light' });
s.add('lm358',    { id:'U1', at:[24,4] });
s.add('resistor', { id:'R1', at:[14,2], value:'10k' });
s.add('vcc',      { id:'P1', at:[24,0], value:'+5V' });
s.add('gnd',      { id:'G1', at:[24,16] });
s.net('+5V', ['P1.+', 'U1.8']);
s.net('IN',  ['R1.2', 'U1.3']);
s.wire('U1.4', 'G1.g');
s.fit(3).render();
const svg = s.toSVG();   // serialized SVG string

8 · Extending the catalog

Drop a self-registering file in /catalog/<category>/<series>/ and list it in catalog/manifest.js:

// catalog/passive/basic/potentiometer.js
SCH.define({
  type:'pot', category:'passive', series:'basic', name:'Potentiometer', ref:'RV',
  w:8, h:3, hidePins:true,
  pins:{ '1':{at:[0,2],side:'l'}, '2':{at:[8,2],side:'r'}, 'w':{at:[4,0],side:'t'} },
  draw:function(p){ p.line(0,2,1.5,2); p.rect(1.5,1.4,5,1.2,{fill:p.theme.body}); p.line(6.5,2,8,2);
                    p.line(4,0,4,1.1); p.polygon([[3.6,1.1],[4.4,1.1],[4,1.6]]); }
});

An IC/module just lists pins per side — pin numbers, names, lead stubs and the body auto-size:

SCH.define({
  type:'ne555', category:'ic', series:'timer', name:'NE555', ref:'U', showName:true, lead:1,
  sides:{ left:['1:GND','2:TRIG','3:OUT','4:RST'],
          right:['8:VCC','7:DISCH','6:THRES','5:CTRL'] },
  draw:function(p){ SCH.icBody(p); }
});

Label placement options (read by the auto-placer at construction):

labelInside: trueallow the designator/name to sit inside the body when sides are crowded — defaults on for showName (IC-style) parts, off for discrete parts
labelPos: 'inside'force inside (e.g. the op-amp triangle)
labelPos: 'right'prefer a side (e.g. transistors, whose top/bottom/left carry pins)

Otherwise the engine auto-picks the least-crowded side around the part.
